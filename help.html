User Guide

Everything you need to manage deliveries with PEP — from importing customers to tracking your drivers in real time.

Getting Started

Follow these five steps to go from a fresh install to your first completed delivery.

1

Add your drivers

Go to Staff Management → Add Staff. Set the role to Driver. You can add as many drivers as you need.

2

Import your customers

Open Current Orders and click Import. Upload a CSV with columns: name, address, postcode, zone. Or add customers one by one using Add Customer.

3

Create a delivery run

Go to Map View and click New Run. Give it a name (e.g. "Monday AM"). Then drag customers from the list into the run, or use Auto-fill to add all customers at once.

4

Optimise the route

Inside the run, click Optimise. The system will sort stops into the most efficient order. You can drag stops to manually reorder them after.

5

Assign a driver and go live

Select a driver from the run panel, then click Start Run. Update each stop as Out for Delivery → Delivered as your driver progresses. Customers receive email notifications at each stage if configured.

Managing Customers

Importing via CSV

Go to Current Orders → Import. Your CSV must have a header row. Supported columns:

postcodeRecommendedSW1A 1AA
zoneNoNorth
dayNoMonday
lat / lngNo51.5074 / -0.1278
If your CSV doesn't have lat/lng, the system will geocode addresses automatically using the postcode or full address.

Adding a customer manually

Click Add Customer in Current Orders. Fill in the name and address, then click Geocode to look up the map coordinates automatically.

Customer Passport

Each customer has a Passport — a record containing their delivery preferences, contact email, and notes. Click the passport icon next to any customer to open it. The email stored here is used for delivery notifications.

Delivery Runs

Creating a run

In Map View, click New Run in the top-right panel. Give the run a name. Runs are usually named by date or time slot — e.g. Monday AM or 2024-06-10 North.

Adding customers to a run

Drag customers from the Customer List on the left into the run panel on the right. Alternatively, right-click a customer on the map and choose Add to Run. Use Auto-fill to add all unassigned customers at once.

Reordering stops

Drag stops up and down inside the run panel to set your preferred delivery order. After optimising you can still drag stops to override the suggested order.

Saving and loading runs

Click Save Run to store the current state. Saved runs appear in the Delivery Runs dropdown at the top of the Map View. Switching runs reloads all stops and route lines for that run.

Route Optimisation

Click Optimise inside a run to automatically reorder stops for the shortest travel time. The route is drawn on the map as a coloured polyline.

Optimisation settings

Go to Settings → OR-Tools to configure how the optimiser behaves:

SettingWhat it does
Max stops per runLimits how many stops one driver can handle
Max distance (km)Caps the total route length
Drop penaltyHow hard the optimiser tries to include every stop
Time limit (s)How long the solver runs before returning its best answer
For large runs (50+ stops), increase the time limit to 30–60 seconds for better results.

Drivers

Driver View

The Driver View screen shows a simplified card-based interface intended for drivers. Each card shows a customer name, address, and status buttons. Drivers tap Out for Delivery when they leave for a stop and Delivered when the order is complete.

Assigning a driver to a run

In the run panel, click the Assign Driver dropdown and select a driver. Once assigned, the driver's name appears on the run header and the route line changes to a solid colour.

Runs with a driver assigned show a solid route line on the map. Unassigned runs show a dotted line.

Adding a new staff member

  1. Click Add Staff
  2. Enter their full name and a username (lowercase, no spaces)
  3. Set a temporary password — they should change it after first login
  4. Select a role
  5. Click Save

Email Notifications

PEP can automatically email customers when their order status changes.

Setup

  1. Go to Settings → Emails
  2. Enter your SMTP server details (host, port, username, password)
  3. Click Send Test Email to verify the connection
  4. Enable the notification types you want to send
  5. Click Save Settings
Using Gmail? You must use an App Password, not your regular Gmail password. Generate one at Google Account → Security → App Passwords.

Which email address is used?

The customer's email is read from their Passport. Open a customer's passport and set the customerEmail field. Without an email address stored there, no notification will be sent for that customer.

Weekly Plan

The Weekly Plan screen gives you a visual timeline of all scheduled deliveries across the week. Each run appears as a block on its assigned day.

  • Drag runs left and right to move them to a different day
  • Click a run block to open it in Map View
  • Use the + Add Run button to schedule a new run directly from this screen

FAQ

How do I reset a customer's status back to Pending?

Click the customer in Current Orders or Map View, change the status to Pending, and save. All email tracking resets automatically so they can receive notifications again in the next run.

Can multiple drivers share one run?

Each run has one assigned driver. To split work between two drivers, create two separate runs and divide the customers between them.

My route optimisation isn't working

Check that the Python OR-Tools service is running (optimise.py) and that the PYTHON_URL environment variable is set correctly. If you're not using OR-Tools, the app falls back to a simple nearest-neighbour sort.

Emails are not being received

  • Check Settings → Emails shows a green Configured badge
  • Use Send Test Email to verify the SMTP connection
  • Confirm the customer's Passport has a valid customerEmail
  • If using Gmail, ensure you're using an App Password, not your account password

How do I add a customer email after import?

Click the passport icon next to the customer's name → add or edit the customerEmail field → Save Passport.

Can customers see their own tracking page?

Not in the current version. Tracking information is communicated via email notifications. A customer-facing tracking page is a planned future feature.
